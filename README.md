# SkyCompass Satellite (人造天体观测罗盘)

**简体中文** | [English](README_EN.md)

## 项目概述

SkyCompass Satellite 是 SkyCompass 项目的扩展演化版本，运行在 M5Stack Cardputer ADV (ESP32) 上。与原项目关注自然天体（太阳、月亮等）不同，Satellite 版本专注于**人造飞行器（如国际空间站 ISS、天宫空间站、哈勃望远镜等）**的实时轨道追踪与可视性预测。

**核心产品定位：** 向用户回答：**“近期有哪些人造飞行器值得观测？”**，它不仅是一个轨道展示工具，更是一个“今晚值得抬头看的飞行器推荐系统”。

## 硬件支持

- M5 Cardputer ADV（ESP32）
- 六轴IMU（加速度计 + 陀螺仪）
- 内置GNSS模块（用于定位及获取高精度 UTC 时间）
- 键盘 + 小尺寸屏幕

## 核心特性

- **精选目标库**：默认内置 ISS、天宫、哈勃等最具观测价值的人造天体，避免因加载海量无效 TLE 数据导致系统卡顿与界面拥挤。
- **实时模式 (Live Mode)**：
  - 纯代码绘制极简地球与简化的大陆轮廓
  - 基于当前太阳位置实时计算并渲染 **昼夜圈** (明暗分界线)
  - 动态展示当前卫星位置、用户定位与简化轨道线（采用非线性高度缩放映射）
- **观测推荐系统 (核心价值)**：
  - 本地解析两行轨道根数 (TLE) 与 SGP4 模型
  - 综合计算：地球遮挡、太阳照明（判断飞行器是否进入地球阴影区）、距离衰减
  - 结合 **亮度预测、最高仰角、持续时间** 自动生成近期最佳观测窗口评分，并输出直观的推荐榜单。
- **本地化离线运行**：全面复用 SkyCompass 的底层 GNSS 和时间系统，轨道推算与可见性判断全部在 ESP32 端离线完成。
- **长期维护能力**：支持本地保存 TLE 文件，内置按需 WiFi 获取最新 TLE 的更新机制。

## 与 SkyCompass 的关系

**共享底层基础：**
- 时间系统与 GNSS 解析
- 坐标转换体系 (ECEF/LatLon/Alt -> Az/Alt)
- 底层 UI 绘制与渲染框架
- 天球计算与太阳位置模型

**业务差异化：**
| 模块   | SkyCompass 原版 | SkyCompass Satellite |
| ---- | ---------- | ---------- |
| 目标   | 太阳/月亮/银河等自然天体       | 人造卫星 / 空间站       |
| 轨道   | 缓慢周期变化        | 极高速变化 (约90分钟绕地球一圈)       |
| 核心体验 | 空间指向与 3D 沉浸观测         | 观测推荐与时间窗口规划       |
| 数据基座   | 天文历表公式       | TLE + SGP4 轨道传播模型 |

## 架构拆解与开发路径

根据项目演进计划，SkyCompass Satellite 将按以下渐进式路径开发：

1. **Phase 0 (架构验证)**：打通 SGP4 模型，实现 TLE 解析与空间坐标解算。
2. **Phase 1 (地球视图 MVP)**：完成极简地球绘制与坐标映射，在屏幕上闭环跑通第一颗卫星的环绕轨迹。
3. **Phase 2 (太阳与光照判断)**：融入昼夜圈逻辑，实现精确的地球阴影遮挡判断（可见性前提）。
4. **Phase 3 (观测推荐系统)**：基于规则引擎（仰角/亮度/时长）输出评分与推荐计划。
5. **Phase 4 (数据管理与脱机运行)**：完善 TLE SD卡读写与 WiFi 按需更新机制。
